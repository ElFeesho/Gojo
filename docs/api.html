Gojo Documentation - 5. API

Gojo has it's own extendable API, though manually extending the API provided is possible, it may be better to extend functionality from within a script rather than altering the Gojo's source, as cross compatability will be lost on other platforms which haven't received the new updates.

Surface Manipulation

surface_load(image filename [, initial x, initial y]) - Returns an 'image' table

Loads 'image filename' and optionally sets the x and y position to an initial x and initial y value.

surface_create(x, y, width, height) - Returns an 'image' table

Creates a blank surface that will be as wide as 'width' and as tall as 'height'. It's position will be 'x' and 'y'.

surface_blit(an 'image' table) - Returns nil

Blits a loaded or created surface onto the screen at it's x and y position. If the image table's alpha value has been altered, the surface's alpha will change accordingly (255 is opaque and 0 is transparent).

Depending on whether Gojo was built to utilise SDL_gfx, if the image table's rot or scale values have been altered, the surface will be blitted at the angle set in the table and scaled respectively based on the factor set in the table.

surface_blit_area(an 'image' table, source x, source y, width, height, destination x, destination y) - Returns nil

Blits the area defined by source x, source y, source width and source height on to the screen at destination x and y.

surface_get_pixel(an 'image' table, x coordinate, y coordinate) - Returns an integer

Checks the pixel at (x, y) on the surface of the image table passed in and returns a value based on the pixels contents.

surface_destroy(an 'image' table) - Returns nil

Frees up the surface reffered to by image table. Image tables contents remain intact, but trying to blit the surface will lead to an error.

Images with freed surfaces are useless and should be made nil.

Drawing Operations

draw_rect(x coordinate, y coordinate, width, height, colour [, border colour]) - Returns nil

Draws a rectangle at x and y, as wide as width and as tall as height. The colour parameter specifies the fill colour. Optionally the rectangle may have a border which can be set with border colour.

draw_line(x coordinate, y coordinate, x2 coordinate, y2 coordinate, colour) - Returns nil

Draws a line from point (x, y) to point (x2, y2) in the specified colour.

Text Manipulation - Dependent on SDL_ttf

text_load(True Type Font filename, Size in PT) - Returns a userdata value

Loads a font file which can be used in other text functions. This function does not scan system font files, so the font file must be in relative directory format. E.g. if the font was in the same directory as the game you would use "./font-name.ttf".

text_draw(x coordinate, y coordinate, font userdata, colour, Text) - Returns nil

Draws text at x and y coordinates using a font loaded with text_load. The text will be in the colour passed in. This function is useful for drawing texts which may vary often.

text_create(x coordinate, y coordinate, font userdata, colour, Text) - Returns an 'image' table

Creates an 'image' table that can be used with surface_blit or surface_blit_area. This function is useful for creating text which will not change through out it's lifetime, e.g. title text, menu texts etc.

text_destroy(font userdata) - Returns nil

Destroys an allocated font, NOT a surface created with text_create. Use surface_destroy to free a surface created with text_create.

Music and Sound manipulation - Dependent on SDL_mixer
